Android-包体积

发表于2023-10-10|更新于2023-10-10|Android
|阅读量:

APK包结构

APK 文件由一个 Zip 压缩文件组成，其中包含构成应用的所有文件。这些文件包括 Java 类文件、资源文件和包含已编译资源的文件。
通常一个APK包含以下目录：

目录 介绍
META-INF/ 签名相关
assets/ 应用程序资源
res/ drawable、layout等资源文件，包含未编译到resources.arsc 中的资源
lib/ 包含特定于处理器软件层的已编译代码，即各种架构的so库
resources.arsc 包含已编译的资源
classes.dex 包含以 Dalvik/ART 虚拟机可理解的 DEX 文件格式编译的类。
AndroidManifest.xml 包含核心 Android 清单文件
kotlin 包含编译后Kotlin的代码

对于resources.arsc : 此文件包含 res/values/ 文件夹的所有配置中的 XML 内容。打包工具会提取此 XML 内容，将其编译为二进制文件形式，并压缩内容。此内容包括语言字符串和样式，以及未直接包含在 resources.arsc 文件中的内容（例如布局文件和图片）的路径。
注意：请勿在 APK 中压缩此文件。

包结构查看方式

现在直接使用Android Studio 自带的Analyzer即可清晰的查看包结构

缩减包体积的方向

由于APK本质也只是一个压缩包，所以减少压缩包体积的本质就是两个方面

  • 减少文件数量
  • 压缩文件大小

缩减包体积的具体方案

删除无用资源文件和代码

通过Android Studio就能过检测无用的资源

但是需要注意的是，该方式是通过Lint工具静态检测的，所以动态引用的资源无法排出，比如: ActivityBinding引用的布局

删除无用的动态库以及动态库分包

  • 动态库一般会占用较多的APK体积，如果没有使用到so库的内容，尽量删除so库。
  • 同时采用分包的形式，目前许多应用市场支持同时上传32位和64位的应用，所以一个应用不必包含多个架构的so库,只保留其中一中即可。

资源尺寸分包

Android平台上的尺寸较多，所以可能某些应用提供了许多套尺寸的资源. 如：hdpi,xhdpi,xxhdpi,xxxhdpi等。
但是目前国内外的手机屏幕尺寸都较为清晰，一般保留其中的xxhdpi即可。

Abdroid App Bundle

利用谷歌推出的一种应用发布格式。
app bundle 针对每种设备配置生成并提供经过优化的 APK，因此只会下载特定设备所需的代码和资源来运行您的应用。所以用户也可以获得更小且更优化的下载文件包。

图片压缩

图片资源在APK的资源也占有比较大的成分，一般可以在不影响展示的最终效果的场景下，可以对其进行压缩

  • 使用Tinypng压缩
  • 使用Android Studio自带的Webp转换工具

结语

包体积随着功能的增加必然是一直增大的，在项目稳定运营之后通常都是需要建立一套监控体系，对应用的体积做好一定的管控
