OC与C语言的关系

OC是C语言的扩展，OC完全兼容C语言

  • OC支持C语言的所有变量类型
  • OC支持C语言的条件语句
  • OC支持C语言的关键字

关键字

OC中除了支持C语言的关键字之外，它还有自己特有的关键字，通常都以@开头

import

import是c语言中#include的加强版

  • include: 每次调用都会拷贝一份代码
  • import : 多次引用同一个文件只会拷贝一次

nill和NULL

nill和NULL意义一样，但是通常区分使用场景

  • nill : OC的类指针
  • NULL : C指针

Foundation框架

Foundation是oc中的基础库，类似c语言的函数库

@autoreleasepool

自动释放池

NSString

NSString是oc特有的字符串类型，用于存放oc中的字符串常量。

static

OC中的static不能修饰类的属性和方法，但是可以在方法内修饰变量，使得变量存储在常量池中。

访问修饰符

点语法

OC可以与Java一样使用.来访问属性，但是必须去掉属性的下划线

点语法在编译期间，会自动调用setter/getter方法

@property

@property的作用就是自动生成对应的getter/setter的声明，但需要注意命名和类型一致。

同时如果手动声明了setter/getter之后，@property不会自动生成其他代码

property是可以携带参数，如：property(参数1，参数2，参数3) ，参数的主要分类有：

  • 与多线程相关的两个参数： atomic、nonatomic
  • 与生成的 setter 方法的实现相关的参数： assign、retain
  • 强弱指针的参数： strong、weak
  • 与生成只读、读写的封装：readwrite、readonly
  • 生成 setter、getter 方法名称相关的参数：setter、getter

id指针和NSObject

  • id指针：在编译期间不会做类型检查

instanceType

代表当前类型对象为当前类，可以作为返回值类型。
