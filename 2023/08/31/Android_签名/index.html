Android-签名相关

发表于2023-08-31|更新于2023-10-09|Android
|阅读量:

APK签名的意义是什么？

官方的说明：
Android 系统要求所有 APK 必须先使用证书进行数字签名，然后才能安装到设备上或进行更新。

个人理解，签名是为了验证APK的唯一性，确认开发者与APK的关系，在升级安装的时候也能正确覆盖安装，避免被恶意同包名的软件覆盖

用什么工具进行签名？

签名的工具为 Java 密钥库（.jks 或 .keystore），是用作证书和私钥存储库的二进制文件。(白话就是密钥库就是存放了证书和私钥的整合包)

jsk和keystore的区别是什么？

密钥库的作用都是用来打包签名的，并保证应用的唯一性，格式的不同只是平台的不同

  • keystore 是 Eclipse 生成的密钥库。
  • jks 是 Android studio 生成的密钥库。

很多第三方市场,我们上传APK的时候,他们只支持keystore,需要我们把.jks签名转化为.keystore!

签名工具的使用场景区分

对于不同的场景，往往需要使用不同的密钥库进行签名，比如

  • debug包使用调试的密钥库 ： 使用的是$HOME/.android/debug.keystore
  • release包使用的密钥库 ： 开发者自己生成的密钥库，可以是 jks 或者 keystore
  • Google Play商店的密钥库 ： 新的应用必须启用 Play 应用签名功能才能使用 AAB 文件。

签名的版本

截止目前2023年，Android签名版本有4个,不同的签名版本签名过程会有所不同

方案
 介绍
v1方案 基于 JAR 签名
签名校验速度慢: 因为需要对所有文件进行摘要
完整性保障不够: META-INF 目录用来存放签名，自然此目录本身是不计入签名校验过程的
v2方案 Android 7.0 引入 是一种全文件签名方案，该方案能够发现对 APK 的受保护部分进行的所有更改，从而有助于加快验证速度并增强完整性保证。解决v1的问题
v3方案 Android 9 支持 APK 密钥轮替，这使应用能够在 APK 更新过程中更改其签名密钥。基于 v2 的升级
APK 签名方案 v3
v4方案 Android 11 通过 APK 签名方案 v4 支持与流式传输兼容的签名方案。
APK 签名方案 v4

v1方案的签名和验证流程

签名流程

  graph LR
A(开始) --对APK所有文件做摘要-->B(.MF)
B --二次摘要---C(*.SF)
C --对*.SF签名，并将签名后的内容与公钥和证书打包--->D(8.RSA)
D --->E(得到签名后的APK)

可以重命名APK为ZIP,并解压，可以得到上面所说的文件

验证流程

  graph TB
A(开始) -->B[安装APK]
B--校验-->C[*.MF]
B--校验-->D[*.SF]
B--校验-->E[*.RSA]

C--通过-->G(安装成功)
D--通过-->G(安装成功)
E--通过-->G(安装成功)

主要包括验证签名、校验完整性两个步骤：

  • 步骤1：验证签名

    • ① 取出*.RSA 中包含的开发者证书，并校验其合法性
    • ② 用证书中的公钥解密*.RSA中包含的签名
    • ③ 用证书中的公钥计算*.SF的签名
    • ④ 对比（2）和（3）的签名是否一致

  • 步骤2：验证完整性

    • ① 检查 APK 中包含的所有文件，对应的摘要值与 MANIFEST.MF 文件中记录的值一致
    • ② 使用证书文件（RSA 文件）检验签名文件（SF 文件）没有被修改过
    • ③ 使用签名文件（SF 文件）检验 MF 文件没有被修改过

上面任何一个步骤验证失败，则整个APK验证失败。

v2方案的签名和验证流程

使用 APK 签名方案 v2 进行签名时，会在 APK 文件中插入一个 APK 签名分块，该分块位于“ZIP 中央目录”部分之前并紧邻该部分。在“APK 签名分块”内，v2 签名和签名者身份信息会存储在 APK 签名方案 v2 分块中。

签名过程


根据官方介绍的大概意思，就是1，3，4部分都会被拆分成多个大小为 1MB（220 个字节）的连续块，然后做数字摘，也就是APK摘要。
然后将 APK 的摘要 + 数字证书 + 其他属性生成签名数据写入到 APK Signing Block 区块。

校验过程

详细可以看官方文档APK 签名方案 v2 验证

V2和V1的兼容方案

在 Android 7.0 及更高版本中，可以根据 APK 签名方案 v2+ 或 JAR 签名（v1 方案）验证 APK。更低版本的平台会忽略 v2 签名，仅验证 v1 签名。
