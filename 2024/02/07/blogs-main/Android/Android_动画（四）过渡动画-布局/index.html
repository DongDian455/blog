过渡动画的定义

过渡动画即“在一个场景到另外一个场景中，能过平滑的切换，让人的观感上体验更佳”。
而在Android系统中，提供了一套完整的过渡动画框架，其主要成员为Transition,Scene,TransitionManager。

而借助框架之后，场景中间的过渡效果就会有框架内部自行实现

过渡动画的优缺点

  • 缺点：有一定使用门槛，在界面切换场景容易出现问题。
  • 优点：框架本身解耦了两个场景之间的连接，通过Transition作为中间者隐藏动画实现内容。

过渡动画源码分析

Scene

Scene的主要成员有上下文，包裹Scene的ViewGroup,以及当前Scene对应的layoutId。所以Scene实际就是记录布局信息。

Transition

Transition是一个抽象类，其内部源码也比较多，但是一般只需要关注的方法为上面的4个:

  • captureStartValues : 记录场景开始的属性值
  • captureEndValues : 记录场景结束的属性值
  • createAnimator : 创建属性动画，也就是真正的过渡动画
  • playTransition : 执行过渡动画的方法

TransitionManager

以过渡动画的执行开始分析

TransitionManager.go需要传递两个参数，一个为切换到的场景Scene,另一个为过渡效果Transition ，接着就是直接调用changeScene方法。

按代码步骤说明：

  • 步骤1 : 判断当前的sceneRoot是否已经执行动画,如果没有才继续进行
    (sceneRoot就是当前包裹scene的外层ViewGroup)
  • 步骤2 : 判断Transition是否为空，如果是空的话直接调用scene.enter()方法。
    （scene.enter() 方法内部，会将sceneRoot的子view全部移除，添加当前scene的view到sceneRoot当中，这样实际已经完成了布局的切换）
  • 步骤3 : 方法内部会调用transition.captureStartValues()，记录动画的开始信息
  • 步骤4 : 调用enter方法，和步骤2的分支一样，会将切换的View添加到当前sceneRoot当中。
  • 步骤5 : 该方法主要是对sceneRoot添加了两个监听，OnAttachStateChangeListener,
    OnPreDrawListener
