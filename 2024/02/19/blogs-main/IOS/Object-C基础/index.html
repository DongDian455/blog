OC与C语言的关系

OC是C语言的扩展，OC完全兼容C语言

  • OC支持C语言的所有变量类型
  • OC支持C语言的条件语句
  • OC支持C语言的关键字

关键字

OC中除了支持C语言的关键字之外，它还有自己特有的关键字，通常都以@开头

import

import是c语言中#include的加强版

  • include: 每次调用都会拷贝一份代码
  • import : 多次引用同一个文件只会拷贝一次

语法

类的定义

OC和C语言一样，声明和定义会进行分开

  • 声明对象

  • 定义对象

  • 实例化

属性定义

属性的声明和类声明一起(也可以放到定义进行声明，这样的话就只有当前类能够访问)，而对象属性需要使用指针*

@property

使用@property声明属性，可以自动生成属性对应的getter/setter方法

如果同时手动声明了setter/getter之后，@property不会自动生成其他代码

property是可以携带参数，如：property(参数1，参数2，参数3) ，参数的主要分类有：

  • 与多线程相关的两个参数： atomic、nonatomic
  • 与生成的 setter 方法的实现相关的参数： assign、retain
  • 强弱指针的参数： strong、weak
  • 与生成只读、读写的封装：readwrite、readonly
  • 生成 setter、getter 方法名称相关的参数：setter、getter

属性访问符

在声明对象的时候，可以给属性添加访问符号
@public,@private,@@protected,@@package

属性一般以下划线为开头，并且声明的时候不能进行初始化

方法声明和定义

方法和类一样，分为声明和定义，和属性一样，如果只实现定义不声明，那么就是只能当钱类访问

OC中的形参数名比较奇怪，它的作用应该是用于描述两个参数之间的关系，而不像swift那样是给参数起一个别名

类方法定义

类方法是独立于类对象的一个方法，类似Java中定义在对象里面的静态方法，只能通过类名直接访问。
普通方法与类方法的区别是方法声明当中的: ‘+’ 和 ‘-‘

nill和NULL

nill和NULL意义一样，但是通常区分使用场景

  • nill : OC的类指针
  • NULL : C指针

Foundation框架

Foundation是oc中的基础库，类似c语言的函数库

@autoreleasepool

自动释放池

NSString

NSString是oc特有的字符串类型，用于存放oc中的字符串常量。

再使用占位符的时候，需要使用的是%@

static

OC中的static不能修饰类的属性和方法，但是可以在方法内修饰变量，使得变量存储在常量池中。

访问修饰符

@public,@private,@@protected,@@package
修饰符的作用域是从变量定义的开始，直接遇到下一个修饰符或者结束括号

点语法

id指针和NSObject

  • id指针：在编译期间不会做类型检查

    id类似于Java泛型中的 * ，就是用于表示任意类型，让编译器能够不检测类型
