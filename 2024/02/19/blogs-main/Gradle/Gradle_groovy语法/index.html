定义变量

使用def关键字就可以定义各类型的变量

定义字符串

  • 单引号(‘’): 定义的字符串为不可扩展字符串
  • 双引号(“”): 定义的字符串为可扩展字符串，可扩展字符串里面可以使用 ${} 引用变量值，当 {} 里面只有一个变量，非表达式时，{}也可以去掉(和kotlin的字符串用法一致)
  • 三引号(‘’’ ‘’’ ): 定义的字符串为输出带格式的不可扩展字符串

Get和Set方法

  1. 创建属性的时候，Groovy会帮我们自动创建 get 和 set 方法
  2. 当我们只定义了一个属性的 get 方法，而没有定义这个属性，默认这个属性只读
  3. 在使⽤对象 object.field 来获取值或者使用 object.field = value 来赋值的时候，实际上会自动转而调⽤ object.getField() 和 object.setField(value) 方法，如果我们不想调用这个特殊的 get 方法时则可以使用 .@ 直接域访问操作符访问属性本身

定义方法和调用

  1. 方法的定义同样使用def
  2. 方法的调用可以不写()
  3. 方法没有返回值可以直接使用void，不过即使没有返回值，默认也会返回object类型的（参考kotlin）
  4. 有返回值类型的时候也可以不写return，默认为最后一行的数据
  5. 类的构造方法，避免添加def关键字

比较

==就相当于 Java 的equals，如果需要比较两个对象是否是同一个，需要使用 .is()

闭包

1、this 永远指向定义该闭包最近的类对象，就近原则，定义闭包时，哪个类离的最近就指向哪个，我这里的离得近是指定义闭包的这个类，包含内部类
2、owner 永远指向定义该闭包的类对象或者闭包对象，顾名思义，闭包只能定义在类中或者闭包中
3、delegate 和 owner 是一样的，我们在闭包的源码中可以看到，owner 会把自己的值赋给 delegate，但同时 delegate 也可以赋其他值
注意：在我们使用 this , owner , 和 delegate 的时候， this 和 owner 默认是只读的，我们外部修改不了它，这点在源码中也有体现，但是可以对 delegate 进行操作
