expect 和 actual

expect主要用于对方法或类的签名前进行声明,actual用于对签名的具体实现

expect 关键字可以修饰类或方法，但是修饰类或方法时，方法都不能有具体实现代码。
expect and actual 声明必须要在同一个包下，有完全相同的限定名。

使用方式

作用

Kotlin官网中说这两个关键字是用来实现跨平台 API 的。具体为expect声明签名，actual实现不同的平台的代码

tailrec

使用方式

标记尾递归函数

作用

在Kotlin中，tailrec是一个关键字，用于标记尾递归函数。尾递归是指在函数的最后一行直接或间接调用自身。
使用tailrec标记的函数可以通过编译器优化为迭代循环，从而避免了递归导致的栈溢出问题，并提高了执行效率。

reified

使用方式

  • 在泛型类型前面增加reified修饰
  • 在方法前面增加inline

作用

泛型在运行的时候会被类型擦除，而在使用inline函数的时候，可以用reified指定类型不擦除或者说变更为具体的类型。

因为inline函数在编译器会将字节码copy到调用它的地方，所以编译器会知道具体的泛型是什么类型，而标记为reified的泛型在编译期就会替换为具体的类型。

internal

使用方式

在类，方法，变量前添加

作用

internal进行修饰的类/方法/变量，表示这个类/方法/变量只适合当前module使用，如果其他module使用的话，会找不到这个internal修饰的目标而报错报错。

typealias

使用方式

与c++类似，用来给变量起一个新的别名

作用

在 Kotlin 中，typealias 是用来为类型定义一个别名的关键字。它可以用于简化复杂的类型并提高代码的可读性。
