Kotlin_协程原理(一)

发表于2024-08-05|更新于2024-09-09|Android
|阅读量:

Continuation

Continuation为协程框架里的核心类，负责接收suspend函数的回调和获取上下文

注：suspend函数本身也是一个Continuation实例

suspend

协程的创建和启动都离不开suspend关键字,它是Kotlin的一个关键字，用于标记当前方法/函数为一个可挂起的。
在complier(编译器)的帮助下，在编译的时候会将suspend修饰的函数添加一个Continuation参数，用于做回调，这样的好处，可以让我们将一些异步代码变成同步的形式，回调的处理由编译器来实现，避免人为查看回调地狱.
另外，编译器还会根据将当前函数，编译为状态模式，根据当前不同的状态，来判断返回结果。

CoroutineContext

CoroutineContext作为协程中的上下文，负责提供资源获取，以及配置管理等工作。

  • CoroutineContext内部定义了Element的获取方法，在实际中每一个上下文当中都持有一个对应的Element对象。
  • CoroutineContext在实际中，通过重载“+”方法，并通过CombinedContext实例来链接每个上下文，类似单链表的形式

Element

  • Element本身又继承上下文，即上下文(CoroutineContext)中持有的元素也是(Element)上下文，这么绕的原因据说是为了方便API的书写和调用。
